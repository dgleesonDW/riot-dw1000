Detailed Description

Interfacing the realtime clock on x86 boards.

Only use this module to read the current time. Using the other functions in applications would break the hwtimer.

◆ x86_init_rtc()

void x86_init_rtc ( void  )

Initialize the Real Time Clock.

This function is called during initialization by x86_startup(). You must not call this function on your own accord.

The RTC subsystem will refuse to work if the CMOS says that

  • there was a power loss,
  • the stored time is invalid (i.e. February 30), or
  • the CMOS had a checksum failure.

◆ x86_rtc_set_alarm()

bool x86_rtc_set_alarm ( const x86_rtc_data_t *  when,
uint32_t  msg_content,
kernel_pid_t  target_pid,
bool  allow_replace 
)

Set an RTC alarm.

Parameters
[in]whenTime when the RTC you raise an interrupt. The date part is ignored.
msg_contentThe value for msg_t::content.value.
target_pidThe process which shall receive the message, KERNEL_PID_UNDEF to disable.
allow_replaceWhether it is allowed to overwrite an existing alarm.

The value of msg_t::type will be reg_c | (RTC_REG_B_INT_UPDATE << 8), where reg_c is the value of CMOS register C.

You should not call this function directly. You should use xtimer instead.
