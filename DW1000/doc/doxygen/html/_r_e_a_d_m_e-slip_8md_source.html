17 ## USB serial port

20 Pin connections for SAMR21 board:
21 
22 * PA23 (RX) -- TXD wire

26 ## Network configuration
27 We will define two networks:
28 
29 1. `bbbb` Between Border Router (BR) and Ubuntu host over TUN
30 2. `aaaa` Between RIOT router node and BR over 6LoWPAN
31 
32 We include two approaches to configuration below, automated via RPL, and manual via fib.
33 
34 First, define a TUN interface on an Ubuntu host with tunslip, in the `dist/tools/tunslip` directory. In the example below, the tun interface is host 1.
35 
36  cd ../../dist/tools/tunslip/
37  sudo ./tunslip6 -s ttyUSB0 -t tun0 bbbb::1/64
38  sudo ip -6 route add aaaa::/64 dev tun0
39 
40 ### RPL-based configuration
41 
42 Configure the interface to the `bbbb` network on the BR from the RIOT terminal. The example commands below assume SLIP is on interface 8.
43 
44  # Set address for SLIP interface
45  ifconfig 8 add unicast bbbb::2/64
46  # Add the Ubuntu host to the neighbor cache
47  ncache add 8 bbbb::1
48 
49 Next configure the interface for the `aaaa` network on the BR. Strictly speaking, it is not necessary to define this address. However, it is convenient for typing, because RIOT automatically creates a more verbose address based on the MAC address for the interface.
50 
51  ifconfig 7 add unicast aaaa::1/64
52  rpl init 7
53 
54 Next, configure the interface on the `aaaa` network on the RIOT router mote.
55 
56  ifconfig 7 add unicast aaaa::2/64
57  rpl init 7
58 
59 Finally, set the BR as the RPL DAG root.
60 
61  rpl root 1 aaaa::1
62 
63 Ping the TUN interface from the router mote, via the BR:
64 
65  ping6 bbbb::1
