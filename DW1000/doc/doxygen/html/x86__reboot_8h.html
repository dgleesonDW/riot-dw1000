typedef void(* x86_reboot_t) (void)

Prototype for a x86 reboot function.

The function may return, then a naive reboot is tried. It is safe to call x86_kbc_reboot() inside the handler, then a naive reboot will be done immediately.

The handler is called with interrupts disabled. You should keep the interrupts disabled and use busy loops.

void NORETURN x86_kbc_reboot ( void  )

Reboots the system.

First the function supplied if x86_set_reboot_fun() is called (iff applies). Then a reboot using the keyboard controller is tried. And if everything fails, a tripple fault is provoked.
