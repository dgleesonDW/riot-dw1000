Compiler dependent defines

These macros use decltype or the earlier __typeof GNU extension. As decltype is only available in newer compilers (VS2010 or gcc 4.3+ when compiling c++ code), this code uses whatever method is needed or, for VS2008 where neither is available, uses casting workarounds.

For VS2008 we use some workarounds to get around the lack of decltype, namely, we always reassign our tmp variable to the list head if we need to dereference its prev/next pointers, and save/restore the real head.
